# 视频合成工作流

## 概述

视频合成将静态分镜图转换为动态视频，支持多种生成模式。

## 视频生成模式

### I2V 模式 (Image to Video)

基于首帧图像生成动态视频。

**输入**:
- 分镜图像（1-7张）
- 视频描述提示词
- 运动指令

**输出**:
- 动态视频片段 (6秒/10秒)

**适用场景**:
- 分镜图转视频
- 静态图像动画化
- 角色动作生成

### R2V 模式 (Reference to Video)

基于参考视频的动作驱动生成。

**输入**:
- 参考视频（角色/场景/道具）
- 目标图像
- 动作描述

**适用场景**:
- 保持角色动作风格
- 场景转换动画
- 复杂动作生成

### T2V 模式 (Text to Video)

纯文本描述生成视频。

**输入**:
- 视频描述提示词
- 画面比例
- 时长

**适用场景**:
- 场景氛围视频
- 过渡镜头
- 特效镜头

## 技术参数

### Grok Imagine 参数

| 参数 | 选项 | 默认值 |
|------|------|--------|
| duration | 6s / 10s | 10s |
| ratio | 2:3/3:2/1:1/16:9/9:16 | 9:16 |
| resolution | 480p / 720p | 720p |
| style | fun / normal / spicy | normal |

### 多Batch抽卡

支持批量生成多个版本供选择：
- 每个分镜可生成多个视频
- 自动保存所有版本
- 方便后续筛选最佳片段

## 生成流程

1. **选择模式** → I2V / R2V / T2V
2. **准备素材** → 图像/视频/文本
3. **构建提示词** → 描述期望效果
4. **调用 Grok API** → 生成视频
5. **等待完成** → 轮询获取结果
6. **下载保存** → 本地保存视频

## 使用方式

### 自然语言调用

- "将分镜图转换为视频"
- "生成动态视频"
- "I2V视频生成"

### 命令行调用

```bash
# I2V 图生视频
python3 scripts/workflow.py generate-video \
  --mode i2v \
  --images frame1.png frame2.png \
  --prompt "镜头推进，人物走向前方" \
  --duration 10

# T2V 文生视频
python3 scripts/workflow.py generate-video \
  --mode t2v \
  --prompt "古风仙侠场景，云雾缭绕" \
  --duration 10 \
  --ratio 9:16
```

## 质量规范

- ✅ 动作流畅自然
- ✅ 符合物理逻辑
- ✅ 风格与分镜一致
- ❌ 无明显抖动/闪烁
- ❌ 无违和感伪影
